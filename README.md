# DocProof Agent

A **hackathon-ready CLI Web3 agent** to register and verify documents on **Avalanche Fuji testnet** using a proof-of-existence smart contract.

The agent stores only **SHA-256 hashes** on-chain, proving a document existed at a given timestamp without storing the document itself.

---

## рџљЂ Project Overview

**DocProof Agent** provides three core user flows:

1. **Register a document** (creates a proof and stores hash on-chain)
2. **Verify a document** (checks on-chain proof for a document hash)
3. **Register a folder** (batch-register many documents)

It also automatically generates a **proof certificate** after registration.

## рџ§  How the CLI Agent Works

### Register a document (`register`)
1. Read the file from disk
2. Generate SHA-256 hash of file contents
3. Check if proof already exists on-chain
4. If not registered, send `storeHash(hash)` transaction
5. Generate a local proof certificate (`proof_certificate_*.txt`)

### Verify a document (`verify`)
1. Read the file
2. Generate SHA-256 hash
3. Call `verifyHash(hash)` on-chain
4. Print owner + timestamp if proof exists

### Register a folder (`register-folder`)
1. List files inside a folder
2. For each file, compute hash + check on-chain
3. Store only new hashes (skips duplicates)
4. Generate certificates for newly registered documents

---

## рџ§© Smart Contract (DocProof.sol)

The smart contract stores a lightweight proof of existence:

- `owner`: address that registered the proof
- `timestamp`: block timestamp when stored

### Contract API

- `storeHash(bytes32 hash)`
  - Stores the proof
  - Reverts with `Document already registered` when the hash is already stored

- `verifyHash(bytes32 hash)`
  - Returns `(exists, owner, timestamp)`

## рџ”Ќ How Document Verification Works

1. The CLI hashes the document using **SHA-256**.
2. The contract stores only the hash (no document data).
3. Later, the same document hash can be computed and checked on-chain.
4. If the hash exists, the contract returns the original registrant and timestamp.

## рџ“ќ Proof Certificate

After a successful registration, the agent creates a local certificate file:

- `proof_certificate_<document>_<timestamp>.txt`

It includes:

- Document name
- Hash
- Owner wallet address
- Transaction hash
- Block timestamp
- Network name
